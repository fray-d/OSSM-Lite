OSSM Lite

3. Choose a homing method

Now that you understand the color codes, and if you followed the advice to not power your motor when flashing the firmware, your device is probably flashing red indicating homing has failed. This was intended. Now you can configure how you want it to home before you power the motor for the first time. Homing fails if it takes longer then 21 seconds to traverse in either direction.

There are four homing types to choose from, and two optional settings. Click the button below to connect to your device, then select the options of your choice. Now's a good time to power your device so the motor can move. Be ready to unplug it if you misconfigure something. Values are saved automatically. Restart to see effect.

Homing Modes

Default: The device will retract to its minimum depth and tap the housing to find home. It will then extend fully to measure the rail before returning again to home. It will move rapidly between points it has previously been.

Double Tap: This is the same as default, except the device will attempt to home inwards twice if it the first tap was very short. This is mostly useful when attaching accessories that might cause an initial home to trigger falsely.

Single Sided: The device will retract to its minimum depth and tap the housing to find home. Instead of measuring, the rail length setting is used. Take care to set it appropriately to avoid crashing.

None: Some devices don't have the required components for homing. This option uses the same user specified value for rail length, but assumes the user has manually parked the rail at home before powering on the device. It will add a safety buffer of 10mm from the initial position. Also useful for developers who want to turn off homing for certain tests.

Optional Settings

Rail Length: This is where you manually specify the length of the rail for modes that require it. The 180mm default works well enough for a standard OSSM. Be sure to leave a safety margin!

Rehoming: You can also set whether the device should home between modes. Setting this will tell the device to return home when switching modes. For IHSV motors this is required for the OSSM to function. For the AIM motors you can disable this feature if you choose.

Requirements: Chrome, Edge, or Opera on desktop (not mobile).
